## 🔴 Blocker (1 item)

### 1. SonarQube token committed in `sonar-project.properties`

**File**: `sonar-project.properties:38`
```
sonar.token=squ_3131a69179595d70506b93facb243d413e973df0
```

The commit `014528f` shows this was present in the original repo and carried into the squash. This is a SonarQube authentication token. Even if the SonarQube instance is local-only, tokens in public repos are bad practice.

**Fix**: Delete the `sonar.token` line from `sonar-project.properties`. Keep the file (useful for self-hosted SonarQube users), just remove the token. If you still use SonarQube, regenerate this token afterward.

```bash
# Remove token from the file (keep rest of config)
sed -i '/sonar.token=/d' sonar-project.properties
git add sonar-project.properties && git commit -m "chore: remove SonarQube token from public repo"
```

**⚠️ Token rotation needed**: After removing from the public repo, rotate the token in your SonarQube instance. Even though history was squashed, the token was findable via GitHub's search before the squash.

## 🟡 Should Fix (6 items)

### 2. Root `package.json` missing standard fields

The root `package.json` is missing: `license`, `repository`, `bugs`, `homepage`. Without these, npm/npx show blank metadata and tools can't auto-link to the repo.

**Fix**: Add to root `package.json`:
```json
"license": "MIT",
"repository": "github:preludeofme/heard-again",
"bugs": "https://github.com/preludeofme/heard-again/issues",
"homepage": "https://heardagain.com"
```

(UI/package.json already has `"license": "MIT"` and `"repository"` — good.)

### 3. RunPod endpoint ID exposed in TTS/RUNPOD_ENDPOINT_SETUP.md

The endpoint ID `8zrnsfsqb8r6u4` is a RunPod endpoint identifier. While not a secret in the traditional sense (it's a public-facing endpoint name), it does reveal infrastructure details. Combined with the endpoint name `heard-again-qwen3-tts-v2`, an attacker could target this specific endpoint.

**Fix**: Replace with placeholder:
```
**Endpoint ID**: `your_runpod_endpoint_id_here`
```

### 4. `UI/next-env.d.ts` tracked in git despite `.gitignore`

The `.gitignore` has `UI/next-env.d.ts` but the file is still tracked: `git ls-files UI/next-env.d.ts` confirms it's in the repo. This is a build artifact that's auto-generated by Next.js and contains dev-time paths.

**Fix**: 
```bash
git rm --cached UI/next-env.d.ts
git commit -m "chore: remove tracked next-env.d.ts build artifact"
```
