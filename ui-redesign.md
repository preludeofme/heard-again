Heard Again — Archive Experience Redesign Prompt

Redesign the application from a multi-section SaaS-style navigation into a single unified “Living Archive” experience optimized for emotional resonance, simplicity, and accessibility for multi-generational users (especially elderly users).

Core Product Shift

This should no longer feel like:

dashboard software
document management
content silos

It should feel like:

entering a loved one’s living archive
walking through a memory space
a digital heirloom

Think Apple-level simplicity, museum exhibit warmth, ancestry-style storytelling, and high trust archival design, not enterprise software.

Primary UX Concept

Move from separate major navigation destinations:

Current:

Timeline
Stories
Documents
Voice Profiles

Redesign into:

Primary Navigation

Use a simplified archive-centric nav:

Archive (default/home)
Contribute
Family Members
Voice Memories
Keepsakes

Remove “Stories” as a primary nav destination.

Stories become a filtered lens inside Archive.

Rename Documents → Keepsakes

Use large click targets and elderly-friendly spacing.

Archive Home (Main Experience)

Create a unified “Person / Family Archive” page.

Hero should feel intimate and elegant.

Include:

Archive Header

Show:

Person/Family name
Lifespan (if applicable)
Family crest/photo/avatar
Archive stats:
Stories
Timeline Events
Voice Memories
Keepsakes
Contributors

Replace separate pages with archive view switching

Below hero create segmented archive modes:

[ Life Journey ] [ Stories ] [ Keepsakes ] [ Voices ]

Large pill or tab controls.

Single archive shell.
Switching modes should feel like changing “lenses,” not navigating to another product.

Default View = Life Journey

This is the centerpiece.

Current timeline should be elevated into signature experience.

Mixed-content memory timeline containing:

milestones
stories
photos

contextual historical moments

Allow filters:

All | Stories | Events | Audio | Keepsakes

Timeline should feel like exploring a life.

Warm, elegant, breathing layouts.
Reduce rigid software-card feeling.

Stories View

Less “audio player cards.”

More:

Elder-Friendly Accessibility Requirements

Optimize specifically for older users:

large typography
high contrast
oversized targets
shallow navigation
minimal decisions
persistent orientation cues
simple labels
no hidden complexity
recognition over recall

Reduce “Where do I go?”
Increase “I understand this instantly.”

Not multiple disconnected sections.
