# Voice Cloning Integration Guide

This guide explains how to use the voice cloning features in the Heard Again application with GPT-SoVITS integration.

## Overview

The voice cloning system allows you to:
1. Upload audio samples of a person's voice
2. Train a custom voice model using GPT-SoVITS
3. Synthesize speech using the trained voice model
4. Use the voice in conversations on the Talk page

## Prerequisites

### Hardware Requirements
- **GPU**: NVIDIA GPU with 6GB+ VRAM (RTX 3060 or higher recommended)
- **RAM**: 16GB+ system memory
- **Storage**: 10GB+ free space for models

### Software Requirements
- Docker & Docker Compose
- NVIDIA Container Toolkit
- CUDA 12.6 or 12.8

## Setup Instructions

### 1. Start GPT-SoVITS Service

```bash
# Start the real GPT-SoVITS service
npm run start:voice:real

# Or start the mock service for testing
npm run start:voice
```

### 2. Start the Main Application

```bash
# In a new terminal
npm run dev
```

### 3. Verify Services

Check that both services are running:
- GPT-SoVITS Web UI: http://localhost:9873
- Main Application: http://localhost:3002

## Using the Voice Cloning Features

### Step 1: Upload Audio Samples

1. Navigate to the **Voice Lab** page
2. Click the **"Train Voice Clone"** button
3. In the modal, upload audio files using the **"Upload Audio Sample"** button
4. Supported formats: WAV, MP3, M4A, FLAC
5. Recommended: At least 1 minute of clear audio

### Step 2: Configure Training

1. Enter a model name (auto-generated by default)
2. Select the language of the audio samples
3. Optional: Configure advanced settings:
   - Noise reduction (recommended)
   - Voice separation (UVR5)
4. Click **"Start Training"**

### Step 3: Monitor Training Progress

The training process has several stages:
1. **Preprocessing** - Audio cleanup and preparation
2. **ASR Processing** - Speech-to-text conversion
3. **Training** - Neural network training
4. **Validation** - Model validation and testing

Training typically takes 10-30 minutes depending on:
- Audio length and quality
- GPU performance
- Server load

### Step 4: Test the Voice

Once training is complete:
1. The model will appear in your voice models list
2. You can test synthesis with any text
3. The voice will be available for use in conversations

## API Endpoints

### Upload Audio Sample
```http
POST /api/voice/upload-sample
Content-Type: multipart/form-data

audio: File
userId: string
```

### Start Training
```http
POST /api/voice/train
Content-Type: application/json

{
  "userId": "string",
  "samples": ["string"],
  "language": "en",
  "modelName": "string"
}
```

### Check Training Status
```http
GET /api/voice/train/{jobId}/status
```

### Synthesize Speech
```http
POST /api/voice/synthesize
Content-Type: application/json

{
  "modelId": "string",
  "text": "string",
  "language": "en",
  "speed": 1.0,
  "pitch": 1.0
}
```

### Get Voice Models
```http
GET /api/voice/models?userId=string
```

## Troubleshooting

### GPT-SoVITS Not Available
```bash
# Check if GPT-SoVITS is running
docker compose -f docker-compose.voice.yml ps

# Check logs
docker compose -f docker-compose.voice.yml logs -f gpt-sovits
```

### GPU Not Detected
```bash
# Verify NVIDIA Docker support
docker run --rm --gpus all nvidia/cuda:12.1.0-base-ubuntu22.04 nvidia-smi
```

### Training Fails
1. Check audio quality - ensure clear, minimal background noise
2. Verify audio duration - at least 60 seconds recommended
3. Check GPU memory usage - may need to reduce batch size
4. Review error logs in the console

### Synthesis Issues
1. Ensure the model training completed successfully
2. Check that the model appears in the models list
3. Verify text is not too long (max 1000 characters)

## Advanced Configuration

### Environment Variables

Create a `.env.local` file:

```env
# GPT-SoVITS Configuration
GPT_SOVITS_URL=http://localhost:9874
GPT_SOVITS_TIMEOUT=300000

# File Storage
UPLOAD_DIR=./uploads
MODEL_DIR=./models
```

### Custom Model Paths

To use pre-trained models or custom model paths:
1. Place models in the `./models` directory
2. They will appear in the models list automatically

## Performance Tips

1. **Audio Quality**: Use high-quality, clean audio recordings
2. **Voice Consistency**: Record all samples in the same session
3. **Background Noise**: Minimize noise for better results
4. **GPU Optimization**: Ensure GPU is not running other heavy tasks
5. **Batch Processing**: Upload multiple samples at once for efficiency

## Security Considerations

- All audio files are stored locally
- Models are isolated per user
- No data is sent to external services
- GDPR compliance features available
